## 시스템 콜

&emsp;&emsp;

>[!info] Definition
> 시스템 콜은 OS에 의해 사용 가능하게 된 서비스에 대한 인터페이스를 제공한다.


&emsp;&emsp;

- 설명
	- 특정 저수준 작업은 어셈블리 명령을 사용하여 작성되어야 하더라도 이러한 호출은 ==**일반적으로 C와 C++ 언어로 작성된 함수 형태로 제공**==된다.

&emsp;&emsp;

### 예제

&emsp;&emsp;

- 설명
	- cp 명령의 경우
		- cli의 경우
			- 시스템 콜
				- 키보드에서 두 파일의 이름을 지정하는 문자을 읽는다.
		- gui의 경우
			- 시스템 콜
				- 화면에서 마우스를 이용해 소스 이름을 선택하여 대상 이름을 지정한다.
			- 특징
				- cli보다 훨씬 많은 I/O 시스템 콜을 필요로 한다.
		- 오류 발생의 경우
			- I/O 시스템 콜이 발생한다.
		- 일련의 동작들이 전부 I/O 시스템 콜이다.

&emsp;&emsp;

### 응용 프로그래밍 인터페이스  - API

&emsp;&emsp;

- 설명
	- OS 별 API 종류
		- Window API
		- POSIX API
			- libc
				- UNIX와 Linux를 위해 C 언어로 작성된 라이브러리
		- Java API (JVM을 OS라고 판단)
	- 특징
		- 간단한 프로그램도 초당 수천개의 시스템 콜이 호출된다.
		- 개발자는 API 명세에 따라 프로그램을 설계한다.
			- OS 인터페이스에 대한 대부분의 자세한 내용은 API에 의해 개발자에게 숨겨지고 RTE에 의해 관리된다.
		- 모든 운영체제는 고유한 시스템 콜 이름을 가진다.
		- API를 구성하는 함수들은 개발자를 대신해서 시스템 콜을 호출한다.
			- win api의 createprocess()는 윈도우 커널의 NTCreateProcess()를 호출한다.
		- 컴퓨터에 따라 시스템 콜은 다른 방법으로 발생한다.
			- 종종 단순히 원하는 시스템 콜이 무엇인지 더 많은 정보가 요구 될 수 있다.
				- OS와 호출에 따라 필요한 정보의 유형과 양은 달라진다.
		- OS가 매개 변수를 전달하는 방법
			- 레지스터 내에 전달
				- 5개 이하의 매개변수가 있으면 사용됨
			- 메모리 내 블록에 저장, 블록의 주소가 레지스터 내 매개변수로 전달
				- 5개를 초과하면 사용됨
			- 메모리 내 테이블에 저장, 테이블의 주소가 레지스터 내 매개변수로 전달
				- 5개를 초과하면 사용됨
			- 프로그램에 의해 스택에 넣어지고 OS에 꺼내질 수도 있다.
				- 일부 운영체제는 블록이나 스택을 선호하는데 매개변수의 길이나 개수를 제한하지 않기 때문
	- RTE
		- 역할
			- 런타임 환경을 의미
		- 특징
			- 컴파일러, 인터프리터를 포함해 특정 프로그래밍 언어로 작성된 응용 프로그램을 실행하는 데 필요한 전체 소프트웨어 제품군과 라이브러리 또는 로더와 같은 소프트웨어를 함께 가리킨다.
			- OS가 제공하는 시스템 콜에 대한 연결 고리 역할을 하는 시스템 콜 인터페이스를 제공한다.
	- 시스템 콜 인터페이스
		- 역할
			- API 함수의 호출을 가로채어 필요한 OS 시스템 콜을 부른다.
		- 특징
			- 각 시스템 콜에는 번호가 할당되어 있고 시스템 콜 인터페이스는 이 번호에 따라 인덱싱된 테이블을 유지한다.
			- 의도하는 시스템 콜을 부르고, 시스템 콜의 상태와 반환 값을 돌려준다.
			- 호출측은 시스템 콜의 구현, 실행 중 무슨 작업을 하는지 아무것도 알 필요 없다.
				- API를 준수하고 시스템 콜의 결과로서 OS가 무슨 일을 할 것인지만 이해하면 된다.
			- 사용자 모드에서 API에 의해 호출된 시스템 콜 인터페이스의 하위 동작들은 커널모드에서 동작되는 과정들이다.
	- 시스템 콜 종류
		- 프로세스 제어
			- 끝내기(end), 중지(abort)
			- 적재(load), 수행(execute)
			- 프로세스 생성, 종료
			- 프로세스 속성 획득, 프로세스 속성 설정
			- 시간을 기다림
			- 이벤트를 기다림, 알림
			- 메모리 할당 및 자유화
		- 파일 조작
			- 파일 생성, 삭제
			- 파일 열기, 닫기
			- 파일 읽기, 쓰기, 위치 변경
			- 파일 속성 획득 및 설정
		- 장치 조작
			- 장치 요청, 해제
			- 읽기, 쓰기, 위치 변경
			- 장치 속성 획득 및 설정
			- 장치의 논리적 부착, 분리
		- 정보 유지 보수
			- 시간과 날짜의 설정과 획득
			- 시스템 데이터의 설정과 획득
			- 프로세스, 파일, 장치 속성의 획득 및 설정
		- 통신과 보호
			- 통신 연결의 생성, 제거
			- 메시지를 송신 및 수신
			- 상태 정보 전달
			- 원격 장치의 부착 및 분리
			- 파일 권한 획득 및 설정
