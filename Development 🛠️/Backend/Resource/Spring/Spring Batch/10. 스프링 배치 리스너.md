# 스프링 배치 이벤트 리스너

&emsp;&emsp;

## 기본 개념

&emsp;&emsp;

### 리스너

&emsp;&emsp;

#### Listeners

&emsp;&emsp;

- 정의
	- 배치 흐름 중 Job, Step, Chunk 단계의 실행 전 후에 발생하는 이벤트를 받아 용도에 맞게 활용할 수 있도록 제공하는 인터셉터 개념의 클래스

&emsp;&emsp;

- 특징
	- 이벤트는 `Listener`를 등록해야 API 설정해서 각 단계 별로 지정할 수 있다.
	- 어노테이션 방식을 사용하면 `Object` 타입의 `Listener`를 등록하게 된다.

&emsp;&emsp;

- 사용 목적
		- 각 단계 별 로그 기록 남기기
		- 소요 시간 계산
		- 실행 상태 정보 참조 및 조회
	

&emsp;&emsp;

- 종류
	- Job
		- `JobExecutionListener`
			- Job 실행 전 후
	- Step
		- `StepExecutionListener`
			- Step 실행 전 후
		- `RepeatListener`
			- Tasklet 실행 전 후
			- 오류 발생 시점
		- `ChunkListener`
			- Chunk 실행 전 후
			- 오류 발생 시점
		- `ItemReadListener`
			- `ItemReader` 실행 전 후
			- 오류 발생 시점
			- Item이 null 일 경우
		- `ItemProcessListener`
			- `ItemProcessor` 실행 전 후
			- 오류 발생 시점
			- item이 null일 경우
		- `ItemWriteListener`
			- `ItemWriter` 실행 전 후
			- 오류 발생 시점
			- item이 null일 경우
	- 그 외
		- `SkipListener`
			- 읽기, 쓰기, 처리 `Skip` 시점
			- Item 처리가 `Skip`될 경우 `Skip`된 item을 추적
		- `RetryListener`
			- `Retry` 시작, 종료, 오류  발생 시점


<br><br><br>

## JobExecutionListener

&emsp;&emsp;

- 정의
	- Job의 실행 전 후 시점에 발생하는 이벤트를 캐치하는 인터셉터

&emsp;&emsp;

- 특징
	- `Job`성공, 실패 여부와 상관없이 호출된다.
	- 성공/실패 여부는 `JobExecution`을 통해 알 수 있다.
	- `JobExecutionListener` 인터페이스 API
		- `void beforeJob(JobExecution)`
			- Job의 실행 전
		- `void afterJob(JobExecution)`
			- Job의 실행 후


<br><br><br>


## StepExecutionListener


&emsp;&emsp;

- 정의
	- Step의 실행 전 후 시점에 발생하는 이벤트를 캐치하는 인터셉터

&emsp;&emsp;

- 특징
	- `Step`성공, 실패 여부와 상관없이 호출된다.
	- 성공/실패 여부는 `StepExecution`을 통해 알 수 있다.
	- `StepExecutionListener` 인터페이스 API
		- `void beforeStep(StepExecution)`
			- Step의 실행 전
		- `ExitStatus afterStep(StepExecution)`
			- Step의 실행 후
			- ==**`ExitStatus`를 변경하면 최종 종료 코드로 반영된다.**==

<br><br><br>

## ChunkListener (RepeatListener)

&emsp;&emsp;

- 정의
	- 트랜잭션 커밋 실행 전 후 및 오류 발생 시점에 발생하는 이벤트를 캐치하는 인터셉터

&emsp;&emsp;

- 특징
	- 오류가 어느 한 읽기 처리 쓰기 구간에서 발생하면, 해당 구간의 에러 이벤트처리 메서드에서 한번, 청크 리스너에서도 에러 이벤트 처리 메서드가 한번 더 호출된다. 
	- `ChunkListener` 인터셉터 API
		- `void beforeChunk(ChunkContext)`
		- `void afterChunk(ChunkContext)`
		- `void afterChunkError(ChunkContext)`

<br><br><br>


## ItemReadListener

&emsp;&emsp;

- 정의
	- `read()`메서드를 실행 전 후 및 오류 발생 시점에 발생하는 이벤트를 캐치하는 인터셉터

&emsp;&emsp;

- 특징
	- `ItemReadListener` 인터셉터 API
		- `void beforeRead()`
		- `void afterRead(T item)`
		- `void onReadError(Exception)`

<br><br><br>

## ItemProcessListener

&emsp;&emsp;

- 정의
	- `doProcess()`메서드를 실행 전 후 및 오류 발생 시점에 발생하는 이벤트를 캐치하는 인터셉터

&emsp;&emsp;

- 특징
	- `ItemProcessListener` 인터셉터 API
		- `void beforeProcess(T item)`
		- `void afterProcess(T item, @Nullable S result)`
		- `void onProcessError(T item, Exception)`

<br><br><br>

## ItemWriteListener

&emsp;&emsp;

- 정의
	- `write()`메서드를 실행 전 후 및 오류 발생 시점에 발생하는 이벤트를 캐치하는 인터셉터

&emsp;&emsp;

- 특징
	- `ItemWriteListener` 인터셉터 API
		- `void beforeWrite(List<? extends S> items)`
		- `void afterWrite(List<? extends S> items)`
		- `void onWriteError(Exception, List<? extends S> items)`

<br><br><br>

## SkipListener

&emsp;&emsp;

- 정의
	- `Skip` 발생 시점에 발생하는 이벤트를 캐치하는 인터셉터

&emsp;&emsp;

- 특징
	- `SkipListener` 인터페이스 API
		- `void onSkipInRead(Throwable t)`
		- `void onSkipInWrite(S item, Throwable t)`
		- `void onSkipInProcess(T item, Throwable t)`

<br><br><br>

## RetryListener

&emsp;&emsp;

- 정의
	- `Retry` 발생 시점에 발생하는 이벤트를 캐치하는 인터셉터

&emsp;&emsp;

- 특징
	- `RetryListener` 인터페이스 API
		- `boolean open(RetryContext, RetryCallback<T, E>)`
			- 재시도 전에 매번 호출된다.
			- false 반환시 retry를 시도하지 않는다.
		- `void close(RetryContext, RetryCallback<T, E>, Throwable t)`
		- `void onError(RetryContext, RetryCallback<T, E>, Throwable t)`
