# 스프링 배치

&emsp;&emsp;

## 개요

&emsp;&emsp;

### 역사

&emsp;&emsp;

- 자바 기반 표준 배치 기술이 부재 했지만 Pivotal과 Accenture라는 기업들의 기여로 탄생하였다.


<br><br><br>


### 배치 핵심 패턴

&emsp;&emsp;

-  설명
	- Read
		- 데이터베이스, 파일, 큐에서 다량의 데이터 조회
	- Process
		- 특정 방법으로 데이터를 가공한다.
	- Write
		- 데이터를 수정된 양식으로 다시 저장한다.

&emsp;&emsp;

#### 추가 설명

&emsp;&emsp;

- 설명
	- ETL
		- 데이터베이스쪽에서 정의한 데이터 웨어하우스 자료 수집 및 이동 기술로 Extract, Transform, Load를 뜻한다.
		- ETL이 배치 핵심 패턴과 비슷하다.


<br><br><br>

### 배치 시나리오

- 설명
	- 배치 프로세스는 주기적으로 커밋될 수 있다.
	- 동시 다발적인 Job의 배치 처리, 대용량 병렬처리가 진행될 수 있다.
	- 실패 후 수동 또는 스케줄링에 의해 재시작될 수 있다.
	- 의존관계가 있는 여러 Step들을 순차적으로 처리될 수 있다.
	- 조건적 Flow 구성을 통한 체계적이고 유연한 배치 모델을 구성된다.
	- 반복, 재시도, skip 처리가 가능하다.


<br><br><br>

### 아키텍처

&emsp;&emsp;

- 설명
	- Architecture Layer 구성
		- Application
			- 스프링 배치 프레임워크를 통해 개발자가 만든 모든 배치 잡과 커스텀 코드를 포함한다.
			- 개발자는 업무로직의 구현에만 집중하고 공통적인 기반기술은 프레임웍이 담당하게 한다.
		- Batch Core
			- Job을 실행, 모니터링, 관리하는 API로 구성
			- JobLauncher, Job, Step, Flow등이 속한다.
		- Batch Infrastructure
			- Application. Batch Core 모두 공통 Infrastructure 위에서 빌드된다.
			- Job 실행의 흐름과 처리를 위한 프레임을 제공한다.
			- Reader, Processor, Writer, Skip, Retry 등이 속한다.
