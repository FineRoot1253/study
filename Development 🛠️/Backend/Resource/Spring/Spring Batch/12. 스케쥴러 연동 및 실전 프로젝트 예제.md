# 스프링 배치 스케쥴러 연동 및 실전 프로젝트 예제

&emsp;&emsp;

## 실전 프로젝트 예제

&emsp;&emsp;

- 요구 사항
	- Job 1
		- 파일로부터 데이터를 읽어 DB에 적재
		- 파일은 매일 새롭게 생성
		- 매일 정해진 시간에 파일을 로드해 데이터를 DB에 업데이트
		- 이미 처리한 파일은 다시 읽지 않는다.
	- Job 2
		- DB로부터 데이터를 읽어 API 서버와 통신
		- 파티셔닝 기능을 통한 멀티 스레드로 구성
		- 제품의 유형에 따라 서로 다른 API 통신을 하도록 구성
		- API 서버는 3개로 구성하여 요청을 처리
		- 제품 내용과 API 통신 결과를 각 파일로 저장
	- 스케줄러
		- 시간을 설정하면 프로그램을 가동
		- 정해진 시간에 주기적으로 Job1, Job2을 실행
		- Quatz 오픈 소스를 활용한다.
